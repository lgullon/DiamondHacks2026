FORMBUDDY — API CONTRACT
=========================
Person A (extension) calls these routes on Person B's FastAPI backend.
Backend runs at: http://localhost:8000
All request/response bodies are JSON unless noted.

DO NOT change field names after agreeing on this — both sides depend on them.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /analyze-form
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Called when: User clicks "Start FormBuddy". Sends all form fields found
on the page. Backend simplifies the questions in the chosen language.

REQUEST:
{
  "fields": [
    {
      "id": "dob",
      "type": "text",
      "label": "Date of Birth",
      "options": null
    },
    {
      "id": "insurance-type",
      "type": "select",
      "label": "Insurance Plan Type",
      "options": ["HMO", "PPO", "EPO", "Medicaid", "Medicare", "No insurance"]
    }
  ],
  "language": "es"
}

RESPONSE:
{
  "questions": [
    {
      "id": "dob",
      "simplified": "¿Cuándo naciste? (ejemplo: 15 de marzo, 1990)"
    },
    {
      "id": "insurance-type",
      "simplified": "¿Qué tipo de seguro médico tienes? Por ejemplo: HMO es cuando necesitas un médico principal, PPO te da más libertad de elegir doctor."
    }
  ]
}


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /voice-input
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Called when: User stops recording. Sends the audio blob for transcription
(Whisper or equivalent). Returns the text of what the user said.

REQUEST: multipart/form-data
  - audio: audio file (webm format from MediaRecorder)
  - language: "en" | "es" | "zh"

RESPONSE:
{
  "transcription": "I was born on March 15th 1990"
}


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /chat
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Called when: Transcription comes back. Claude decides if the user gave
a real answer (ready to fill) or needs a follow-up clarification.

REQUEST:
{
  "field_id": "dob",
  "user_message": "what does date of birth mean?",
  "language": "es"
}

RESPONSE (when user needs clarification — loop back to recording):
{
  "reply": "Es la fecha en que naciste, por ejemplo: 15/03/1990.",
  "ready_to_fill": false,
  "fill_value": null
}

RESPONSE (when user gave a real answer — fill the field):
{
  "reply": "Perfecto, voy a escribir eso ahora.",
  "ready_to_fill": true,
  "fill_value": "03/15/1990"
}


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /speak
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Called when: Backend sends a reply text that needs to be spoken aloud.
Returns an audio file (ElevenLabs or equivalent TTS).

REQUEST:
{
  "text": "¿Cuándo naciste?",
  "language": "es"
}

RESPONSE: audio file (binary)
  Content-Type: audio/mpeg  (or audio/wav)

NOTE: If this route is slow to implement, the extension has a fallback
using the browser's built-in speechSynthesis API — so Person B can
stub this last and it won't block Person A.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
POST /fill-field   (Person B triggers this via Browser Use)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Called when: /chat returns ready_to_fill: true. Tells the backend to
use Browser Use to actually fill the form field in the browser.

REQUEST:
{
  "field_id": "dob",
  "value": "03/15/1990"
}

RESPONSE:
{
  "success": true
}

LANGUAGE CODES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"en" = English
"es" = Spanish
"zh" = Chinese (Mandarin)


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CORS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Person B MUST enable CORS on the FastAPI server or the extension
will be blocked by the browser. Add this to main.py:

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
